### 1. USERS (Anti-Duplikasi âœ…)
localStorage key: "users"
- Validasi: Nama & Email UNIK
- ID: Auto-generate dengan timestamp
- Fields: id, nama, email, status, password, isAdmin, tanggalDaftar

### 2. BOOKS (Panel Admin)
localStorage key: "books"
- Auto sync ke dashboard
- Fields: id, title, author, year, description, cover, quantity

localStorage key: "peminjaman"
- Mencatat setiap peminjaman
- Status: "Dipinjam" atau "Dikembalikan"
- Fields: id, nama, buku, tanggalPinjam, status, tanggalDaftar

### 4. PENGEMBALIAN (Return Records)
localStorage key: "pengembalian"
- Mencatat pengembalian buku
- Fields: id, nama, buku, tanggalPengembalian, tanggalDaftar

## âš ï¸ CATATAN PENTING

1. LOCAL STORAGE LIMITATIONS:
   - Limit kapasitas ~5-10MB per domain
   - Persistent sampai cache browser dihapus
   - Client-side only (tidak tersimpan di server)

2. SECURITY CONCERNS:
   - Password disimpan plaintext (bukan hash)
   - Data terlihat di browser DevTools
   - UNTUK PRODUCTION: gunakan backend + bcrypt + HTTPS

3. BACKUP:
   - Selalu backup data sebelum update
   - Gunakan db.download() untuk backup JSON
   - Test di browser berbeda jika diperlukan

4. TROUBLESHOOTING:
   - Jika data hilang: cek localStorage di DevTools
   - Jika duplikasi: clear cache & refresh
   - Jika error: cek console F12 untuk error messages
